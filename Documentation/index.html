Getting Started

Thanks for purchasing Snapify!

Here, we've put together a comprehensive documentation to help you out using snapify - and making your life easier.

What is Snapify and Who is it for?

Snapify is a WordPress plugin that lets you backup and move your entire WordPress site to another server easily.

This is done by packing your whole site (themes, plugins, settings, content, etc...) into a setup wizard that you can just throw onto your new server and install.

On your new server you just access the PHP installer and it will automatically restore all of your website - you won't even need to enter the new URL!

Snapify is built from 2 separate parts, one is the WordPress plugin and one which is the installer.

The Installer File

The installer is a script which is supposed to restore your WordPress site after backupping with the Snapify plugin.

The installer is not part of the Snapify WordPress plugin - it is only used as a file for the download of the backup so you could restore that backup.

Which means it does not follow the WordPress standards - it has to include jquery by itself because when it is used WordPress is not installed yet.

This file is not part of the WordPress plugin, but is used only after backupping, to restore your site.

Who is it for?

Snapify is great for:

  • regular site admins, who need to backup their site or move it to a new hosting provider.
  • site developers, when you create a local demo of the site you can easily move it to production with Snapify.

Usage

Requirements

  • PHP 5.2.4+ (mbstring extension or zip extension)
  • MySQL 5.0+
  • WordPress 3.0 +
Known Issues
  • Does not handle multi-site as one website yet.
  • Does not work with cloudflare at the moment.
  • Does not work with hostings that have a low php execution time that can’t be changed
  • Please contact us before buying if you think you have one of these problems

Install

To install Snapify, open the WordPress admin panel upload, install and activate the Snapify Plugin.

Then you can click on: Tools->Snapify there press Backup.

Then you will get a download prompt of a zip file. This is your whole site's' backup.

You will have to use the installer download link to download the installer, without that file you can't install the backup on your new server

To move your site on a new location upload the backup zip file and extract it on your new server. Make sure that the files have write and execute permissions.

Then browse to the snapify.php file and press Install. You need to enter your new MySQL settings (DB name, user, pass and host), after that it will install and direct you to your site - and you're all done!

Common Questions and Problems

Make sure you are using the latest version of snapify before contacting support.

Does it support multisite?

Not currently, but please tell us if this is a feature you need.

Debug file not found!

Did you antecedently delete the debug log file? if not, Please contact us with your debug log.

Error connecting to MySQL server: ...

Please check that your credentials are correct and that the supplied settings in the installer for MySQL are correct.

Bad character set detected...

This may be resolved if you use the "Force-UTF8" checkbox in the Snapify plugin (when backupping) and try again.

Database name supplied is already created...

This means that MySQL already has a database with the name you supplied. You can override it by checking the "Override Database" checkbox when installing. Or, delete the database manually.

Could not create database...

This means that Snapify can't create a new database either because you didn't use the root MySQL user or because you didn't grant the user the ability to create a new database. If you can't grant that permission, then you can create the database yourself and click the "Override Database" checkbox when installing.

Error selecting MySQL database: ...

Double check that your user has the required permissions for the supplied database name.

Halting import, error performing query ...

Make sure that you didn't edit the snapify.sql file, because this means that it has SQL errors, please contact us with your debug log, so we could help you out.

Error performing query ...

Please contact us with your debug log.

The current directory isn't writable ...

Check that the Snapify direcotry has write permissions for the web user (usually www-data). If not, you can run: [chmod -R 777 ./] in the directory, keep in mind that this isn't the best security practice, so change it after snapify finishes.

Failed opening required ... wordfence-waf.php ...

This means that your Wordfence WAF is blocking the restore of your website because it uses bad paths. Please disable it and try to backup again without it.

What are all these "Backup Methods"?

Compressed Download - will compress your files on the server and then initiate a backup process

Stream Download - will compress your files on the fly and start the download imminently (good for large sites, that don't have space)

Database Backup Only - will not include all of your files, but will include your database contents. Then you can add your files yourself (via ftp/scp download) and add them to the snapify.zip, used mainly in debug situations.
